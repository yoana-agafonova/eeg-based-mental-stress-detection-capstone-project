	% ABSTRACT
	% =====================================================
	\begin{abstract}
		This report presents a data science project for EEG-based mental stress detection. The aim of the project is to classify EEG recordings into stress and non-stress states using machine learning. The dataset consists of EEG recordings collected under several experimental conditions, including complex mathematical problem solving, Trier mental challenge testing, Stroop colour word testing, horror video stimulation, and relaxing music listening. The stress-inducing conditions were grouped into the stress class, while the relaxing music condition was used as the non-stress class.
		
		The methodology includes data loading, selection of relevant EEG channels, signal preprocessing, window-based segmentation, feature extraction, and model evaluation. Five EEG channels were used: AF3, T7, Pz, T8, and AF4. The EEG signals were filtered using a 0.5--45 Hz bandpass filter and divided into 10-second windows. For each window, statistical features and frequency-band power features were extracted. Several machine learning models were compared, including Logistic Regression, Random Forest, Support Vector Machine, K-Nearest Neighbors, and Gradient Boosting.
		
		The results show that window-based classification improves the amount of available training data, but evaluation must be performed carefully because windows from the same recording can be highly similar. Therefore, GroupKFold validation was used to keep windows from the same EDF recording together. Under this stricter evaluation, Gradient Boosting achieved the best Macro F1-score. The final model performed well in detecting stress windows, but struggled with the minority non-stress class, mainly due to class imbalance. Overall, the project demonstrates that EEG features can be used for stress detection, while also highlighting the challenges of imbalanced biomedical signal classification.
	\end{abstract}

	\section{Introduction}
	
	\subsection{Problem Statement}
	
	Mental stress is an important factor affecting health, cognitive performance, productivity, and general well-being. In many situations, stress is assessed through questionnaires or self-reports. Although these methods are easy to use, they are subjective and may not always reflect the actual physiological state of a person. Because of this, physiological signals such as electroencephalography (EEG) can provide a more objective way of analyzing stress-related changes.
	
	This project focuses on building a machine learning classifier for mental stress detection using EEG recordings. The task is formulated as a binary classification problem, where EEG signal windows are classified as either stress or non-stress. The stress class includes recordings from mentally demanding or emotionally stimulating tasks, while the non-stress class includes relaxed-state recordings.
	
	\subsection{Objectives}
	
	The main objectives of this project are:
	
	\begin{itemize}
		\item to explore and understand the provided EEG dataset;
		\item to define stress and non-stress classes based on the experimental conditions;
		\item to preprocess EEG signals and select relevant EEG channels;
		\item to extract statistical and frequency-domain features from EEG windows;
		\item to train and compare several machine learning models;
		\item to evaluate the models using suitable metrics for imbalanced classification;
		\item to analyze the limitations and possible improvements of the final model.
	\end{itemize}
	
	\subsection{Dataset Overview}
	
	The dataset used in this project was provided through a soft copy link from Mendeley Data. It contains EEG recordings stored as EDF files and organized by experimental condition. The available conditions include complex mathematical problem solving, horror video stimulation, Stroop colour word testing, Trier mental challenge testing, and relaxing music listening.
	
	The selected EEG channels used in this project are AF3, T7, Pz, T8, and AF4. The recordings have a sampling frequency of 128 Hz. The four task-based conditions were grouped into the stress class, while the relaxing music condition was used as the non-stress class.
	
	\subsection{Background and Challenges}
	
	EEG data is a challenging type of biomedical signal because it can contain noise, artifacts, and strong individual variability between subjects. In addition, the dataset used in this project is imbalanced, with more stress recordings than non-stress recordings. This affects model training and evaluation because a model may achieve high accuracy by mainly predicting the majority class. For this reason, balanced accuracy and Macro F1-score are used in addition to standard accuracy.

	% 2. LITERATURE REVIEW
	% =====================================================
	\section{Literature Review}

	Mental stress detection using physiological signals has been an active research area for several decades. Early foundational work by Healey and Picard \cite{healey2005} demonstrated that physiological signals, including skin conductance, heart rate, and electromyography, can reliably distinguish stress states during real-world driving tasks. Their work established that objective, sensor-based measurements offer a meaningful improvement over self-reported assessments, which are inherently subjective and susceptible to recall bias.

	Electroencephalography has since become a widely used modality for stress monitoring because it captures direct neural correlates of cognitive and emotional states. A review by Alberdi et al.~\cite{alberdi2016} systematically examined physiological and behavioral signals for automatic stress recognition and highlighted EEG as one of the most informative channels, particularly when combined with frequency-domain features. More recently, Saeed et al.~\cite{saeed2020} proposed a classification framework for long-term stress using psychological labeling and EEG recordings, achieving competitive results through frequency band power extraction followed by traditional machine learning classifiers.

	A central finding across the EEG stress detection literature concerns the relevance of specific frequency bands. Studies consistently report that the alpha band (8--13~Hz) and beta band (13--30~Hz) are the most informative for stress classification. Increased beta power and decreased alpha power are associated with heightened cognitive arousal and stress, while theta activity (4--8~Hz) has been linked to emotional regulation \cite{bandratios2021}. The gamma band (30--45~Hz) has received less attention in traditional machine learning approaches, though it is associated with high-level cognitive processing.

	The choice of machine learning models for EEG-based stress classification varies across the literature. Logistic Regression, Support Vector Machines, Random Forests, and Gradient Boosting classifiers have all been applied with varying success \cite{stressreview2022}. SVM with radial basis function kernels is frequently used due to its ability to find non-linear decision boundaries in high-dimensional feature spaces. Ensemble methods such as Random Forest and Gradient Boosting have shown competitive performance, particularly when the training set is large enough to avoid overfitting. Gradient Boosting in particular builds an additive model in a stage-wise fashion, as described by Friedman \cite{friedman2001}, and has proven effective for tabular feature data in biomedical settings.

	A common challenge in the EEG stress detection literature is evaluation validity. Many studies use random cross-validation without accounting for the fact that multiple recordings may originate from the same subject or session. This leads to data leakage and inflated performance estimates. Studies such as the review by Stuldreher et al.~\cite{stuldreher2021} emphasize the importance of subject-independent validation to ensure that models generalize to new individuals rather than memorizing session-specific artifacts.

	Based on the literature, the expected performance of a well-designed EEG stress classifier under subject-independent or file-independent evaluation is typically in the range of 70--85\% accuracy and 0.55--0.75 Macro F1-score for binary stress/non-stress classification. These values reflect the inherent difficulty of the problem due to inter-subject variability, class imbalance, and EEG artifact contamination. These benchmarks serve as our success criteria for this project.

	% =====================================================
	\section{Methodology}

	\subsection{Data Collection and Description}

	The dataset used in this project was obtained from Mendeley Data \cite{dataset} and consists of 112 EDF files organized into five experimental condition folders. Each EDF file contains a multi-channel EEG recording captured with an EMOTIV EPOC 5-channel sensor kit at a sampling frequency of 128~Hz. The recordings vary in duration across subjects and sessions.

	Four conditions were treated as stress: Complex Mathematical Problem Solving (CMPS, 22 files), Horror Video Stimulation (22 files), Stroop Colour Word Test (SCWT, 24 files), and Trier Mental Challenge Test (TMCT, 24 files). The relaxing music condition (20 files) was used as the non-stress class. This labeling strategy follows the established approach in the literature, where task-induced cognitive load or emotional arousal is treated as the stress condition, and passive relaxation as the non-stress condition.

	The EDF files contain 31 channels in total, including timestamp channels, quality indicators, battery channels, and marker channels. For this project, only the five EEG channels were used: AF3, T7, Pz, T8, and AF4. These channels cover frontal (AF3, AF4), temporal (T7, T8), and central parietal (Pz) regions and represent a balanced spatial coverage given the hardware constraints of the EMOTIV device.

	\subsection{Problem Formulation}

	The task is formulated as a binary classification problem. Let $\mathbf{x} \in \mathbb{R}^{75}$ represent a feature vector extracted from a 10-second EEG window, where 75 features are derived from five channels with 15 features per channel (10 statistical features and 5 frequency-band power features). The target label $y \in \{0, 1\}$ denotes non-stress (0) or stress (1). The goal is to learn a classifier $f: \mathbb{R}^{75} \rightarrow \{0, 1\}$ that generalizes to unseen EDF recordings.

	\subsection{Signal Preprocessing}

	All EEG signals were processed using the MNE Python library \cite{mne}. For each EDF file, the five EEG channels were selected and then filtered with a finite impulse response bandpass filter in the range of 0.5--45~Hz. The lower cutoff at 0.5~Hz removes slow DC drift and baseline wander artifacts, while the upper cutoff at 45~Hz removes high-frequency noise and power-line harmonics. No artifact rejection or independent component analysis was applied, given the scope of this project.

	\subsection{Window-Based Segmentation}

	Rather than extracting one feature vector per EDF file, each recording was divided into consecutive, non-overlapping 10-second windows. At 128~Hz, each window contains 1280 samples per channel. A window is only included if the full 10 seconds of data are available, so the last incomplete segment is discarded. This windowing approach increases the total number of samples from 112 EDF files to 3507 windows, which is essential for training machine learning models.

	The resulting class distribution after windowing is 3037 stress windows (86.6\%) and 470 non-stress windows (13.4\%). This significant imbalance reflects the imbalance at the recording level (92 stress files vs.\ 20 non-stress files) and has important implications for model evaluation.

	\subsection{Feature Extraction}

	For each 10-second window and each of the five EEG channels, 15 features were extracted, giving a total feature dimensionality of 75. The features are divided into two groups:

	\textbf{Statistical features (10 per channel):} mean, standard deviation, variance, minimum, maximum, median, skewness, kurtosis, signal energy (mean squared amplitude), and Shannon entropy computed from the signal histogram.

	\textbf{Frequency-band power features (5 per channel):} the power spectral density was estimated using Welch's method \cite{welch1967}, and the average power was computed by integrating the PSD using the trapezoidal rule over five standard EEG frequency bands: delta (0.5--4~Hz), theta (4--8~Hz), alpha (8--13~Hz), beta (13--30~Hz), and gamma (30--45~Hz).

	\subsection{Model Training and Evaluation}

	Five classical machine learning classifiers were evaluated: Logistic Regression, Random Forest (300 estimators), Support Vector Machine with RBF kernel, K-Nearest Neighbors (k=5), and Gradient Boosting \cite{sklearn}. Each model was wrapped in a scikit-learn \texttt{Pipeline} that applies \texttt{StandardScaler} normalization before classification. Class weighting (\texttt{class\_weight=`balanced'}) was applied to Logistic Regression, Random Forest, and SVM to partially compensate for the class imbalance.

	Two cross-validation strategies were used. First, a 5-fold \texttt{StratifiedKFold} was applied to windows randomly split across folds. This approach is optimistic because windows from the same EDF recording can appear in both training and test folds, introducing data leakage. Second, a 5-fold \texttt{GroupKFold} was applied with groups defined at the EDF-file level, ensuring that all windows from one recording are assigned to a single fold. This is the stricter and more realistic evaluation strategy.

	The evaluation metrics used are: accuracy, balanced accuracy, and Macro F1-score. Balanced accuracy and Macro F1-score are preferred for imbalanced classification because they treat both classes equally. Accuracy alone is misleading when the majority class dominates.

	The final model was selected based on GroupKFold Macro F1-score. A group-based \texttt{GroupShuffleSplit} with 20\% test size was used for the final hold-out evaluation to obtain the classification report and confusion matrix.

	\subsection{Tools and Software}

	The entire workflow was implemented in Python 3. The main libraries used are: MNE \cite{mne} for EDF loading and EEG signal preprocessing, NumPy and SciPy for numerical computations and spectral analysis, pandas for data management, scikit-learn \cite{sklearn} for machine learning pipelines, cross-validation, and metrics, and Matplotlib for visualization.

	Table~3 and Figure~\ref{fig:model_comparison_group} summarize the cross-validated performance of all five models under 5-fold GroupKFold validation, where all windows from a given EDF file are kept in the same fold. 	\subsection{Final Model Performance}

	The final Gradient Boosting model was evaluated on a held-out test set of 714 windows (20\% of the full dataset, split by EDF file using \texttt{GroupShuffleSplit}). Table~4 shows the full classification report and Figure~\ref{fig:confusion_matrix} shows the corresponding confusion matrix.

	The model achieves an overall accuracy of 0.82 and a Macro F1-score of 0.58, which meets the success criterion of Macro F1 $\geq$ 0.55 defined in Section~2. However, non-stress recall is only 0.16 — the model correctly classifies just 23 out of 141 non-stress windows. This asymmetry is a direct consequence of the severe class imbalance: the model has seen approximately 6.5 times more stress windows during training and defaults to predicting stress in most cases. The non-stress F1-score of 0.25 confirms that minority class detection remains the primary weakness of the system.

	% =====================================================
	\section{Discussion}

	\subsection{Interpretation of Results}

	The results reveal a clear and important gap between the two cross-validation strategies. Under random \texttt{StratifiedKFold}, Random Forest achieved a Macro F1-score of 0.86, which appears highly competitive. However, once the evaluation was corrected using \texttt{GroupKFold} to prevent windows from the same recording from appearing in both train and test folds, the best Macro F1-score dropped to 0.59 (Gradient Boosting). This large gap -- over 0.27 F1-score -- is a direct consequence of data leakage. Windows extracted from the same EDF file are highly correlated because they capture the same subject under the same condition. A model trained on some windows from a file and tested on others from the same file is essentially memorizing recording-specific signal characteristics rather than learning a general stress classifier.

	The GroupKFold results represent the true generalization capacity of the models. Under this stricter evaluation, all five models perform substantially worse than their random-CV counterparts, and the differences between models narrow considerably. Gradient Boosting ranked first with Macro F1 = 0.59, followed closely by KNN (0.56) and SVM (0.56). SVM with RBF kernel achieved the highest balanced accuracy (0.64) among all models, suggesting it has the best trade-off between sensitivity to both classes, even if its overall accuracy is lower.

	\subsection{Class Imbalance and Non-Stress Detection}

	The most significant challenge revealed by the final model evaluation is the failure to detect non-stress windows reliably. The final Gradient Boosting model achieved a recall of only 0.16 for the non-stress class, meaning it correctly identified only 16\% of non-stress windows. This is a direct consequence of the severe class imbalance (86.6\% stress vs.\ 13.4\% non-stress). Even though class weighting was applied to some models, Gradient Boosting does not natively support it in scikit-learn without manually adjusting sample weights, which was not applied in this project. The model learned to predominantly predict the stress class, achieving high accuracy (0.82) but a very low Macro F1-score (0.58).

	This is a well-known problem in biomedical signal classification and was anticipated from the dataset distribution. Techniques such as SMOTE (Synthetic Minority Over-sampling Technique) or cost-sensitive learning could directly address this limitation in future work.

	\subsection{Feature Importance}

	The feature importance analysis from the final Gradient Boosting model reveals that gamma-band power from the AF4 channel is by far the most important feature, accounting for approximately 26\% of total importance. This finding is somewhat surprising, as the alpha and beta bands are typically highlighted in the stress detection literature. The prominence of gamma power could reflect high-frequency cognitive engagement during the stress tasks, but it could also indicate that some high-frequency artifacts were not fully removed by the 45~Hz filter cutoff. Several other frequency-domain features from T8, T7, Pz, and AF4 also appear in the top 15, alongside the skewness and kurtosis of the AF3 channel, which are sensitive to non-Gaussian amplitude distributions.

	\subsection{Limitations}

	Several limitations of this work should be acknowledged. First, the class imbalance was not addressed with advanced resampling techniques, which likely degraded non-stress recall significantly. Second, no artifact rejection or independent component analysis was performed, meaning that eye movements, muscle artifacts, and electrode noise may have influenced the extracted features. Third, the Gradient Boosting model was used with default hyperparameters, and no hyperparameter optimization was performed. Finally, the problem setting aggregates four different stress-inducing conditions into a single class, which may introduce within-class heterogeneity that complicates learning.

	\subsection{Possible Improvements}

	Future improvements to this work could include applying SMOTE or other oversampling strategies to address class imbalance, performing artifact removal before feature extraction, tuning model hyperparameters through nested cross-validation, and exploring subject-level generalization by splitting data strictly by subject identity. Deep learning approaches such as convolutional or recurrent neural networks operating directly on raw EEG windows could also be explored as an alternative to handcrafted feature extraction. Additionally, extending the feature set with connectivity measures between channels or non-linear features such as sample entropy and Hjorth parameters could improve classification performance.

	% 6. CONCLUSION
	% =====================================================
	\section{Conclusion}

	This project investigated the feasibility of detecting mental stress from EEG recordings using classical machine learning classifiers. Starting from raw EDF files containing five EEG channels, we designed a complete data science pipeline that includes signal preprocessing, window-based segmentation, statistical and frequency-domain feature extraction, model training, and rigorous cross-validation.

	The central methodological contribution of this project was the deliberate comparison of two validation strategies. The random \texttt{StratifiedKFold} evaluation produced inflated results due to data leakage between highly correlated windows from the same EDF recording. The \texttt{GroupKFold} evaluation corrected this by enforcing strict recording-level separation between training and test data, yielding a more honest estimate of model generalization. This finding is directly relevant to objective 6 of the project and highlights a methodological pitfall that is common in EEG-based classification literature.

	Among the five models evaluated, Gradient Boosting achieved the best Macro F1-score of 0.59 under GroupKFold validation, satisfying the project's objective of identifying the most suitable model. The final model detected stress windows with high recall (0.98) and reasonable precision (0.83), but struggled significantly with non-stress window detection (recall = 0.16), which is attributable to the 86.6\%/13.4\% class imbalance in the dataset. Feature importance analysis showed that gamma-band power, particularly at the AF4 channel, dominated the model's decision, alongside contributions from frequency features across multiple channels.

	In summary, the project demonstrates that EEG features extracted from short time windows contain sufficient information to distinguish stress from non-stress states at a level above random chance, even under conservative evaluation conditions. However, the class imbalance problem remains a critical limitation that must be addressed in future work. Applying oversampling techniques, artifact rejection, and subject-independent evaluation protocols are the most promising directions for improving both the robustness and the clinical applicability of the approach.
